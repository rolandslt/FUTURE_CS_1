# Vulnerability Assessment Report – Live Website (Read-Only)

## Overview

This project performs a **read-only vulnerability assessment** on a public website to identify common security weaknesses and provide clear remediation recommendations.

## Objective

* Analyze a public website for security risks
* Classify vulnerabilities (Low / Medium / High)
* Explain issues in simple business language
* Suggest practical fixes

## Scope

**Allowed**

* Public pages analysis
* Passive scanning
* Security header inspection
* Configuration review

**Not Allowed**

* Exploitation or attacks
* Login bypass attempts
* Brute force or DoS testing

## Tools Used

* Nmap (exposed services analysis)
* OWASP ZAP – Passive Scan
* Browser DevTools (headers, cookies)
* Canva ([report design](https://www.canva.com/design/DAHFRN4xoY4/DPXciN6EXgfOso-yP07nUA/edit?utm_content=DAHFRN4xoY4&utm_campaign=designshare&utm_medium=link2&utm_source=sharebutton))

## Methodology

1. Selected a public website
2. Performed passive security checks
3. Reviewed open services and headers
4. Identified visible weaknesses
5. Classified risk severity
6. Documented findings and fixes



## Risk Levels

* **Low** – Minor improvement needed
* **Medium** – Potential security risk
* **High** – Serious issue requiring attention
