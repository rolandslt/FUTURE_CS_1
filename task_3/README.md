# API Security Risk Analysis

## Overview

This project demonstrates a **read-only API Security Risk Analysis** on public/demo APIs to identify common security risks and recommend improvements, similar to a SaaS security assessment.

## APIs Tested

* https://jsonplaceholder.typicode.com

## Tools Used

* Postman / Insomnia
* Browser Developer Tools
* Documentation (PDF report)

## Methodology

1. Reviewed API documentation
2. Tested endpoints (GET / safe requests)
3. Inspected headers and responses
4. Checked authentication and access control
5. Identified risks and assigned severity
6. Suggested remediation steps

## Security Areas Reviewed

* Unauthenticated endpoints
* Excessive data exposure
* Authentication & authorization
* Rate limiting
* Input validation

## Risk Severity

* **Low** – Minor issue
* **Medium** – Potential risk
* **High** – Serious security concern
